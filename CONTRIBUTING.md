# Contributing

## Development

1. Fork the repository and create a focused branch.
2. Do not add real accounts, Cookie values, databases, logs, API keys or personal paths.
3. Keep behavior compatible with the existing FastAPI and React structure.
4. Add focused tests for behavior changes.
5. Run the checks documented in `README.md`.

## Commits

Use Conventional Commits, for example:

```text
feat(ai): add product-scoped training rules
fix(settings): preserve an existing account key
docs: clarify skill capability limits
```

## Pull Requests

Describe the root cause, user impact, implementation and verification. For UI changes, include desktop and mobile evidence without exposing account or order data.
